Running

To run R you need to add the correct module to your environment. 

module add R

You should make sure that you have set the environment variable $TMP to point to a directory in your /work filesystem before running R. e.g. 

export TMP=/work/z01/z01/aturner/temp

Example job submission script

As R runs in serial you will either have to run your jobs on the serial queues or use SPRINT or your own MPI task farm which you can then use with aprun. (You must also ensure that the R script keeps track of the results in a sensible way - e.g. by writing to task specific files.) The following example uses SPRINT.

An R script that calls SPRINT. Save as 'install_test.R'. 

library("sprint")
ptest()
pterminate()
quit()

The job submission script. Save as 'submission_script.pbs'.

#!/bin/bash --login
#PBS -N R_job
#PBS -l mppwidth=32
#PBS -l mppnppn=32
#PBS -l walltime=01:00:00
# Replace with your own buget
#PBS -A b10                 

#module add R

# Replace with your own temporary directory.
export TMP=/home/z01/z01/egrant1/work/Rinstall/tmp  

export R_WORK_HOME=/usr/local/packages/rpackage/2.15.2-phase3/R-2.15.2

# Change to the directory that the job was submitted from
cd $PBS_O_WORKDIR

aprun -n 4 -N 4 $R_WORK_HOME/bin/R -f install_test.R

Submit the code with the command:

 qsub submission_script.pbs

Check the queue:

 qstat -u $USER

The output of the results should be in R_job.o{job_no.} and R_job.e{job_no.}
